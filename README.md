# 💼 Jomble — Job Application Tracker

Jomble is a sleek, modern, and intuitive Job Application Tracker designed to help developers and job seekers organize, track, and analyze their job search progress in one place. Built with Next.js (App Router), React 19, Tailwind CSS (v4), Prisma, and Auth.js.

---

## ✨ Features

- 📊 **Analytics Dashboard**: Get instant insights into your job search with key metrics:
  - Total applied & weekly trends
  - Response rate and average reply time calculations
  - Ghosted rate tracking
  - Visual status pipeline funnel
- 🗂️ **Job Management**: Complete CRUD operations for your applications. Track company name, job title, location, job URLs, work mode (Remote/Hybrid/On-site), salary range (supporting IDR, USD, SGD), excitement score, and status.
- 🕒 **Timeline Tracking**: Keep a chronological record of your application progress (e.g., status updates, interview schedules, note additions).
- 📝 **Notes & Contacts**: Keep detailed notes and recruiter/hiring manager contacts associated with each application.
- 🔑 **Authentication**: Secure credentials-based authentication and Google OAuth integration via Auth.js (NextAuth v5).
- 🌓 **Themes**: Fully responsive UI with seamless Light/Dark mode transitions.

## 🚀 Getting Started

### Prerequisites

Ensure you have the following installed on your machine:
- Node.js (v18.x or later)
- npm, pnpm, or yarn
- A PostgreSQL database instance (Supabase, Neon, or local database)

### Setup Instructions

1. **Clone the repository**:
   ```bash
   git clone https://github.com/your-username/jomble-app.git
   cd jomble-app
   ```

2. **Install dependencies**:
   ```bash
   npm install
   ```

3. **Configure Environment Variables**:
   Copy the example environment variables file and fill in your details:
   ```bash
   cp .env.example .env
   ```
   Modify `.env` with your Postgres connection string and authentication secrets:
   - `DATABASE_URL`: Your PostgreSQL database URL
   - `AUTH_SECRET`: Generate a secret by running `npx auth secret` or `openssl rand -base64 33`
   - `GOOGLE_CLIENT_ID` & `GOOGLE_CLIENT_SECRET`: Credentials from the Google Cloud Console (if using Google Login)

4. **Initialize the Database**:
   Run the Prisma migration to set up database schemas:
   ```bash
   npx prisma migrate dev
   ```

5. **Seed the Database (Optional)**:
   Populate the database with a test user and 100 realistic mock applications:
   ```bash
   npx tsx prisma/seed.ts
   ```
   *Demo credentials generated by seed:*
   - **Email**: `demo@jomble.com`
   - **Password**: `password123`

6. **Start the Development Server**:
   ```bash
   npm run dev
   ```
   Open [http://localhost:3000](http://localhost:3000) in your browser to view the application.
